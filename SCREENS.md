---

## SC-001 · 로그인

**목적:** 이메일·비밀번호로 인증하고 역할(STAFF / SUPERVISOR)에 따라 초기 화면으로 이동한다.

### 구성 요소

| 요소 | 유형 | 설명 |
|---|---|---|
| 이메일 | Text Input | 필수 |
| 비밀번호 | Password Input | 필수 |
| 로그인 버튼 | Button | 인증 실행 |
| 오류 메시지 | Alert | 인증 실패 시 표시 |

### 동작

- 로그인 성공 → SC-002(일일 보고서 목록)로 이동
- 인증 실패 → 오류 메시지 인라인 표시, 입력 유지

---

## SC-002 · 일일 보고서 목록

**목적:** 보고서를 날짜 기준으로 탐색한다. STAFF는 본인 보고서만, SUPERVISOR는 전체 보고서를 조회한다.

### 구성 요소

| 요소 | 유형 | 설명 |
|---|---|---|
| 월 선택 | Month Picker | 기본값: 당월 |
| 담당자 필터 | Dropdown | SUPERVISOR만 표시. 전체 / 개인 선택 |
| 보고서 목록 | Table | 날짜, 작성자, 방문 건수, 작성 상태, 조작 버튼 |
| 작성 상태 뱃지 | Badge | 작성완료(초록) / 미작성(회색) |
| 새 보고서 작성 버튼 | Button | STAFF만 표시. 오늘 보고서가 없을 때 활성화 |

### 보고서 목록 컬럼

| 컬럼 | 내용 |
|---|---|
| 보고 날짜 | YYYY-MM-DD |
| 작성자 | 영업사원 이름 (SUPERVISOR 뷰) |
| 방문 건수 | VISIT_RECORD 행 수 |
| 상태 | 작성완료 / 미작성 |
| 액션 | 상세 보기 / 편집(본인·당일만) |

### 화면 전환

- 행 클릭 → SC-004(상세)
- 새 보고서 작성 버튼 → SC-003(작성)
- 편집 버튼 → SC-003(편집 모드)

---

## SC-003 · 일일 보고서 작성·편집

**목적:** STAFF가 당일 보고서를 작성하거나 수정한다.

### 구성 요소

#### 헤더 영역

| 요소 | 유형 | 설명 |
|---|---|---|
| 보고 날짜 | Date (읽기 전용) | 작성: 오늘 / 편집: 해당 날짜 고정 |
| 작성자 | Text (읽기 전용) | 로그인 사용자 이름 |

#### 방문 기록 섹션

| 요소 | 유형 | 설명 |
|---|---|---|
| 방문 기록 테이블 | Editable Table | 행별로 고객 + 방문 내용 입력 |
| 고객 | Searchable Select | CUSTOMER 마스터에서 선택 |
| 방문 내용 | Textarea | 필수 |
| 행 추가 버튼 | Button | 빈 행 1개 추가 |
| 행 삭제 버튼 | Icon Button | 행 단위 삭제 (최소 1행 유지) |

#### Problem 섹션

| 요소 | 유형 | 설명 |
|---|---|---|
| Problem | Textarea | 현재 과제·상담 내용 자유 입력 |

#### Plan 섹션

| 요소 | 유형 | 설명 |
|---|---|---|
| Plan | Textarea | 내일 할 일 자유 입력 |

#### 하단 버튼

| 요소 | 유형 | 설명 |
|---|---|---|
| 저장 버튼 | Primary Button | 임시 저장(언제든 재편집 가능) |
| 취소 버튼 | Text Button | SC-002로 복귀 |

### 유효성 검사

- 방문 기록: 고객과 방문 내용 모두 필수
- 방문 기록 행은 최소 1개

### 화면 전환

- 저장 완료 → SC-004(상세)로 이동

---

## SC-004 · 일일 보고서 상세

**목적:** 보고서 내용을 열람하고, SUPERVISOR는 Problem·Plan에 댓글을 남긴다.

### 구성 요소

#### 헤더 영역

| 요소 | 유형 | 설명 |
|---|---|---|
| 보고 날짜 | Text | |
| 작성자 | Text | |
| 편집 버튼 | Button | STAFF 본인 + 당일 보고서일 때만 표시 |

#### 방문 기록 섹션

| 요소 | 유형 | 설명 |
|---|---|---|
| 방문 기록 테이블 | Table (읽기 전용) | 고객명, 방문 내용 |

#### Problem 섹션

| 요소 | 유형 | 설명 |
|---|---|---|
| Problem 본문 | Text | 작성된 내용 표시 |
| 댓글 목록 | List | 작성자, 작성일시, 내용 |
| 댓글 입력 폼 | Textarea + 등록 버튼 | SUPERVISOR만 표시 |

#### Plan 섹션

| 요소 | 유형 | 설명 |
|---|---|---|
| Plan 본문 | Text | 작성된 내용 표시 |
| 댓글 목록 | List | 작성자, 작성일시, 내용 |
| 댓글 입력 폼 | Textarea + 등록 버튼 | SUPERVISOR만 표시 |

### 동작

- SUPERVISOR가 댓글 등록 → 해당 섹션 댓글 목록 즉시 갱신
- 편집 버튼 → SC-003(편집 모드)

## SC-005 · 고객 마스터 목록

**목적:** 등록된 고객을 검색하고 관리한다.

### 구성 요소

| 요소 | 유형 | 설명 |
|---|---|---|
| 검색창 | Text Input | 회사명 또는 담당자명으로 검색 |

| 편집 버튼 | Icon Button | SUPERVISOR만 표시 |

### 화면 전환

- 신규 등록 버튼 → SC-006(등록 모드)
- 편집 버튼 → SC-006(편집 모드)
